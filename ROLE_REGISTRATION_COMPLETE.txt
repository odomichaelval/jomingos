================================================================================
                            HOW IT WORKS (FLOW)
================================================================================

User Registration Flow:

1. USER VISITS REGISTRATION PAGE
   Backend: http://localhost:8000/accounts/register/
   OR
   Frontend: http://localhost:3000/register

2. USER FILLS IN FORM
   • Username
   • First Name
   • Last Name
   • Email
   • PASSWORD WITH ROLE SELECTION ← NEW
   • Confirm Password

3. USER SELECTS ROLE FROM DROPDOWN ← KEY FEATURE
   ┌─────────────────────────┐
   │ Select your role...     │
   │ ─────────────────────── │
   │ Care Assistant          │
   │ Nurse                   │
   │ Doctor                  │
   │ Administrator           │
   │ Family Member           │
   └─────────────────────────┘

4. USER CLICKS "CREATE ACCOUNT"

5. BACKEND VALIDATES
   ✓ Username unique?
   ✓ Email unique?
   ✓ Password meets requirements?
   ✓ Role valid? ← ROLE VALIDATION

6. ACCOUNT CREATED WITH ROLE
   User record saved with selected role

7. JWT TOKENS ISSUED
   • Access token
   • Refresh token

8. RESPONSE INCLUDES REDIRECT PATH
   {
     "user": {...},
     "access": "JWT_TOKEN",
     "refresh": "REFRESH_TOKEN",
     "dashboard_path": "/dashboard/nurse"  ← Based on selected role
   }

9. FRONTEND AUTO-REDIRECTS
   If Nurse selected: → /dashboard/nurse
   If Doctor selected: → /dashboard/doctor
   If Care Assistant selected: → /dashboard/care-assistant
   (etc.)

10. ROLE-SPECIFIC DASHBOARD LOADS
    User sees content tailored to their role

================================================================================
                              API ENDPOINT
================================================================================

POST /api/accounts/register/

REQUEST EXAMPLE:
{
  "username": "john_nurse",
  "email": "john@example.com",
  "first_name": "John",
  "last_name": "Smith",
  "password": "SecurePass123!",
  "password2": "SecurePass123!",
  "role": "nurse"  ← ROLE PARAMETER (NEW)
}

SUCCESS RESPONSE (201):
{
  "user": {
    "id": 1,
    "username": "john_nurse",
    "email": "john@example.com",
    "role": "nurse",
    "full_name": "John Smith"
  },
  "access": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
  "refresh": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
  "dashboard_path": "/dashboard/nurse"
}

ERROR RESPONSE (400):
{
  "email": ["This email is already registered."],
  "role": ["Invalid role selected."],
  "password": ["Password must contain special characters"]
}
